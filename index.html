PulseClass — Live classroom engagement, simplified

Class project proposal • Lightweight web app to run interactive polls, quizzes, and attendance for live classes
Status: Proposal
Phase: MVP
Last updated: October 22, 2025

Application Overview

From an end-user perspective, PulseClass is a lightweight web app instructors open in class to quickly launch polls, short quizzes, and attendance checks that students answer from any device — no install required. Instructors see real-time results and simple analytics. Students get immediate feedback and can review their responses later.

Feature List (Prioritized)

  1. MVP Polls & Live Results — Instructor creates a poll, students respond, live results chart updates.
  2. Short Quizzes (Multi-choice) — 1–5 question quizzes with instant scoring and answer reveal.
  3. Student Join via Code/Link — Short session code or link; no account required for students.
  4. Attendance Tally — Quick attendance check with automatic tally per session.
  5. Instructor Dashboard — Start/stop sessions, view aggregate responses, export CSV.

Prioritized by deliverability in a typical semester-length class project.

Future Feature List (Non-prioritized)

  • Live chat for students (moderated)
  • Rich question types: free-response, image-based, code-run snippets
  • Student accounts and progress tracking across sessions
  • Integrations: Canvas/LMS, Google Classroom single-sign-on
  • Mobile PWA with offline caching and push notifications

Basic Technical Features (what we learned in class)

  • Responsive HTML/CSS layout (semantic elements, accessible headings)
  • Client-side JavaScript for dynamic UI and real-time updates (WebSockets / polling)
  • REST API endpoints for session and result management
  • Relational data modeling for sessions, questions, responses (simple CRUD)
  • Version control (Git) and deployment pipeline (simple host like GitHub Pages or Netlify for front-end)

Advanced Technical Features (beyond class)

  • Real-time updates via WebSocket server (Socket.IO or native WebSockets)
  • Stateless JWT-based auth for instructors + optional role-based access
  • Containerized deployment with Docker + simple CI (GitHub Actions) to build and run tests
  • Horizontal scaling hints: ephemeral WebSocket session store + Redis pub/sub
  • Data privacy: opt-in analytics, anonymized storage for research

Hosting & Deployment

Quick hosting options for a class project:

  • Frontend-only / static: GitHub Pages or Netlify — push `index.html` or a build output from React.
  • Full stack / backend: Render, Heroku (free/low-tier dev plan), or Fly.io for containers. Use GitHub Actions for CI to run tests and deploy.
  • Socket / real-time: Ensure the host supports WebSocket connections or use a dedicated real-time host (e.g., Fly.io, Render, or a small VPS).
